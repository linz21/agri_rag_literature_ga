# Trimmed for HF Spaces deployment — inference only, not the full dev/eval

# `spaces` is required for the @spaces.GPU decorator used by ZeroGPU

# 2.8.0 as of this deployment) — the Space's own "Configuration error"

# This is DIFFERENT from the local dev environment's torch==2.3.0 (see
# main project's requirements.txt) — that's fine, these are separate
# execution environments with separate constraints.
#
# gradio is intentionally NOT pinned here — README.md's sdk_version field
# is the correct mechanism Hugging Face uses to control which Gradio
# version gets installed for a Gradio-SDK Space. Having it ALSO in
# requirements.txt was redundant and likely contributed to a
# gradio_client/pydantic schema bug (TypeError: argument of type 'bool'
# is not iterable) hit during initial deployment.
